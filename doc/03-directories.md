# ディレクトリ構成の解説

プロジェクト作成後のディレクトリ構成は下記のようになっています。

```
.
|-- app         (クラスファイル置き場)
|   |-- action  (アクションスクリプト)
|   |-- action_cli  (CLI用アクションスクリプト)
|   |-- plugin  (プラグインフィルタスクリプト)
|   |-- test    (テストスクリプト)
|   `-- view    (ビュースクリプト)
|-- bin         (CLIスクリプト)
|-- etc         (設定ファイル等)
|-- lib         (サードパーティライブラリ)
|-- locale      (国際化用メッセージカタログ)
|   `-- ja_JP
|-- log         (ログファイル)
|-- schema      (DBスキーマ等)
|-- skel        (スケルトンファイル)
|-- template
|   `-- ja_JP   (テンプレートファイル)
|-- tmp         (一時ファイル)
`-- www         (ウェブ公開ディレクトリ)
    |-- css     (CSSファイル)
    `-- js      (JavaScriptファイル)
```

たくさんありますが、必ずしもこれら全てを使う必要はありません。

開発で頻繁に使うのは下記のディレクトリです。

* app
* app/action
* app/view
* template/ja_JP
* www
